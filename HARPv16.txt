v16 → v17 Roadmap

The following items close the remaining gaps between "certifiable prototype" and "series-production SIL 3 device":

SOEM no_std port — replace the EtherCAT DC write stub with a real no_std EtherCAT master; add Prusti DMA-overrun proofs on the ipc.rs unsafe blocks.

Prusti DMA proofs — formally prove that a DMA overrun in ipc.rs / hal_nk.rs cannot corrupt the ESKF covariance matrix (Prusti #[requires(dma_ring_is_valid())] contract).

Full HJ grid — generate the offline HJ value function for the actual deployment environment via helperOC / hj_reachability Python toolchain and flash to the ITCM .hj_table section.

Full SLAM integration — replace the HlcIntent stub with a complete ROS2/Zenoh bridge consuming Nav2 costmap edges over the TEE-validated UART channel.

Learned gait classifier — replace threshold-based gait phase detection with a 4 kB quantised biomechanical model running on the STM32H7 DSP core.

TrustZone attestation — add a full attestation chain from the TEE monitor to an external TPM, enabling remote audit of the firmware safety-state history.

Secure boot — verify the firmware Merkle root against an OTP-fused expected root at init, completing the chain of trust from hardware to application.

Creusot loop invariants (full) — add Creusot-style loop invariants to the Merkle batch flush and conformal sort passes for fully deductive, unbounded proofs (no unrolling required).

MC/DC automation — integrate cargo-llvm-cov --mcdc into the CI pipeline as a mandatory gate; generate the MC/DC report automatically and inject coverage percentages into the RTM PDF via build.rs.

Fleet analytics — ship deterministic Merkle digests (with incline) to an offline analytics backend for per-user MobilityPolicy recalibration.
